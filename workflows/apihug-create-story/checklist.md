## ApiHug Forbidden Rules Validation (CRITICAL)

> **If ANY item below is violated, story MUST be corrected before proceeding to dev-story**

### Proto Design Core Constraints
- [ ] **Pageable API:** Request Message has NO page/size/sort fields (uses `pageable: true`)
- [ ] **Pageable Response:** NEVER wrap in `PageableXxx` (framework handles `Result<Pageable<T>>`)
- [ ] **Request Message:** NOT created for <3 fields (uses `google.protobuf.Empty` + `parameters{}`)
- [ ] **Enum Values:** NEVER use `string` for enums (use `EnumType` with full package path)
- [ ] **File-Level Options:** NO `java_package`, `go_package`, `java_multiple_files`
- [ ] **Cross-Layer Import:** API layer does NOT import domain entities
- [ ] **SQL Reserved Words:** Column names NOT in reserved list (user→user_name, order→order_no)
- [ ] **Table Prefix:** Table names have prefix: SYS_, BIZ_, CFG_, LOG_
- [ ] **Type References:** NEVER relative references (use full package: `com.example.Order`)
- [ ] **DTO Naming:** NEVER use `DTO` suffix in message names
- [ ] **Inner Messages:** NEVER define inner message types (all at file level)
- [ ] **ENUM Suffix:** NEVER redundant "ENUM" suffix (`STATUS_ACTIVE` ✅)
- [ ] **Migration File:** NEVER modify `_full.xml` (auto-generated by wire)
- [ ] **Enum Persistence:** NEVER use `ORDINAL` (use `STRING` or `CODE`)
- [ ] **Duplicate Codes:** NEVER duplicate `code` values within same enum
- [ ] **Enum Field Values:** NEVER numeric values (use enum names: `http_status: NOT_FOUND`)

### Feature Package Organization
- [ ] **Feature Subdirectories:** Each feature has subdir in `api/`, `domain/`, `infra/`
- [ ] **API Package:** `{proto_api_path}/{feature}/{feature}_service.proto` + `{feature}_dto.proto`
- [ ] **Domain Package:** `{proto_domain_path}/{feature}/{feature}_entity.proto`
- [ ] **Infra Package:** `{proto_infra_path}/{feature}/{feature}_constant.proto` + `{feature}_error.proto`
- [ ] **NO Root Files:** NEVER place proto files directly in `api/`, `domain/`, `infra/` root

### Implementation Forbidden Rules
- [ ] **Query Logic:** NO query logic in ServiceImpl (ALL in trait repository)
- [ ] **Validation:** NO manual proto-constrained field validation in ServiceImpl
- [ ] **RBAC:** NO @PreAuthorize, @Secured, or manual permission checks
- [ ] **Result Builder:** NEVER call `builder.done()` (controller handles it)
- [ ] **Error Handling:** NEVER throw RuntimeException for business errors

### Source Set Forbidden Rules
- [ ] **Generated Code:** NEVER modify src/generated/ (READ-ONLY)
- [ ] **ServiceImpl:** NEVER create manually (run `gradlew :module:wire`)
- [ ] **Trait Repository:** NEVER create manually (auto-generated by wire)

## ApiHug Golden Rules Compliance

### Proto Design Rules (if story involves proto)

#### API Extension ({apihug_proto_api_extension_guide})
- [ ] **Pageable API:** Request has NO page/size/sort; uses `pageable: true`; Response is SINGLE item type
- [ ] **Request Message Design:** 0 params → Empty; 1-2 params → Empty+parameters{}; 3+ params → Message
- [ ] **Swagger Extensions:** service/rpc/message/field annotations defined
- [ ] **Cross-Layer:** API layer does NOT import domain entities
- [ ] **Authorization:** Uses `low_limit_risky_mode` or `rbac` in proto (NOT manual code)
- [ ] **Proto DSL:** ALL fields use snake_case; NEVER inner message types

#### Database Modeling ({apihug_proto_database_modeling_guide})
- [ ] **SQL Reserved Words:** Column names NOT in reserved list; Table has prefix (SYS_, BIZ_, CFG_, LOG_)
- [ ] **Enum Persistence:** Uses `STRING` or `CODE` (NEVER `ORDINAL`)
- [ ] **DB Migration:** Mentions `_full.xml` auto-generated; corrections use `_after.xml` with `preConditions`

#### Enum/Error Extension ({apihug_proto_enum_error_extension_guide})
- [ ] **Error Codes:** Unique codes within domain
- [ ] **Error Fields:** Uses http_status/phase/severity enums (NOT numbers/strings)
- [ ] **Enum Definition:** code, message required; message2 optional; error sub-message for errors

### Implementation Rules (if story involves backend)
- [ ] **ServiceImpl:** Notes that file is partially generated; NEVER modify comments/signatures
- [ ] **Repository Trait:** Query logic in trait (`t.*Repository`), NOT in ServiceImpl
- [ ] **Error Handling:** DomainService throws `HopeErrorDetailException`; ServiceImpl uses `builder.error()`
- [ ] **Pagination:** Receives `PageRequest` from framework; 0-based index; NEVER call `builder.done()`
- [ ] **Security:** ZERO RBAC code in ServiceImpl/DomainService — proto authorization only
- [ ] **Lombok:** FORBIDDEN — explicit getters/setters/constructors required
- [ ] **Source Set:** NEVER modify `src/generated/` — comments are wire markers

### Frontend Rules (if story involves Vue pages)
- [ ] **Auto-Router:** NEVER manually create router files — generated by `gradlew :ui_module:ui`
- [ ] **API SDK:** Import from `@scope/module-sdk` (NEVER write raw HTTP/fetch/axios)
- [ ] **defineOptions:** Unique name (e.g., SystemUser); meta has title, icon, authority
- [ ] **Page Patterns:**
  - LIST_PAGE: `views/{module}/index.vue` with `useVbenVxeGrid`
  - FORM_PAGE: `views/{module}/components/{Form}Modal.vue` with `useVbenForm` + `useVbenModal`
  - DETAIL_PAGE: `views/{module}/[id]/index.vue` with `hideInMenu: true`
- [ ] **Vben Components:** Correct usage of `useVbenVxeGrid`, `useVbenForm`, `useVbenModal`
- [ ] **Page Index:** 0-based conversion (`currentPage - 1` for vxe-table)
