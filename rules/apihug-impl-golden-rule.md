# ApiHug Implementation Golden Rule

> **Onion Architecture**: Adapter (ServiceImpl) → Domain (DomainService optional) → Infrastructure (Repository)

---

## MANDATORY EXECUTION RULES (READ FIRST):

- 🛑 FORBIDDEN remove existing class-level documentation; appending new content is permitted, but the original must remain intact
- 🛑 **ZERO RBAC code** in ServiceImpl/DomainService/Repository — framework AOP handles auth
- 🛑 **NEVER** modify class-level comments in ServiceImpl/Trait — wire markers for regeneration
- 🛑 **NEVER** move/rename ServiceImpl or Trait Repository — breaks wire tracking
- 🛑 **NEVER** write query logic in ServiceImpl — ALL queries in Repository trait
- 🛑 **NEVER** call `builder.done()` in ServiceImpl — controller handles response
- 🛑 **NEVER** use `@PreAuthorize`, `@Secured`, `@RolesAllowed` — proto-driven RBAC only
- 🛑 **NEVER** throw `RuntimeException` for business errors — use `HopeErrorDetailException`
- 🛑 **NEVER** manually validate proto-constrained fields — framework validates automatically
- 🛑 **NEVER** modify `src/generated/` files — READ-ONLY auto-generated code
- 🛑 **NEVER** use field injection (`@Autowired`) — use constructor injection
- ✅ **ALWAYS** run `gradlew :{module}:wire` before implementing — generate skeletons
- ✅ **ALWAYS** use `grep` to locate generated files — `grep -r "class XxxServiceImpl" src/main/java/`
- ✅ **ALWAYS** put query logic in Repository trait (`t._*Repository`) — separation of concerns
- 🛑 **NEVER**  manually create any class in the `src/main/trait`, they are all partial generated by apihug.
- ✅ **ALWAYS** throw `HopeErrorDetailException.errorBuilder(ENUM).build()` in DomainService
- ✅ **ALWAYS** use `builder.error(ENUM)` or propagate exception in ServiceImpl
- ✅ **ALWAYS** use 0-based page index — Spring Data convention
- ✅ **ALWAYS** filter soft-delete: `deleted = false` — data integrity
- ✅ **ALWAYS** use `exists()` instead of `findOne().isPresent()` — performance
- ✅ **ALWAYS** add pagination for list queries — prevent unbounded results
- 🛑 **NEVER** paginate in memory (`subList`/`skip`/`limit` on full result) — ALWAYS use DB-level pagination
- ✅ **ALWAYS** use `HopeContextHolder.customer()` for current user — security context

## ⚠️ CRITICAL: Partially Generated Files — Immutable Skeleton

> **Applies to**: `ServiceImpl` and `Trait Repository` (`t.*Repository`)

### What "Partially Generated" Means

| File Type            | Location                                                        | Generation Behavior                                                                         |
|----------------------|-----------------------------------------------------------------|---------------------------------------------------------------------------------------------|
| **ServiceImpl**      | `src/main/java/{package}/api/{Service}ServiceImpl.java`         | `wire` generates **class skeleton + method signatures** → Developer fills **method bodies** |
| **Trait Repository** | `src/main/trait/t/{package}/domain/repository/*Repository.java` | `wire` generates **interface skeleton** → Developer adds **custom query methods**           |

### Absolute Rules (ZERO TOLERANCE)

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│  PARTIALLY GENERATED FILES — IMMUTABLE RULES                                   │
├─────────────────────────────────────────────────────────────────────────────────┤
│  ❌ NEVER move the file to another directory                                   │
│  ❌ NEVER rename the file or class                                             │
│  ❌ NEVER manually create the file yourself                                    │
│  ❌ NEVER delete or modify class-level comments (they are wire markers)        │
│  ❌ NEVER change method signatures generated by wire                           │
│  ✅ ONLY fill in method bodies with business logic                             │
│  ✅ ONLY add new methods to trait repository interfaces                        │
│  ✅ If file not found → run `gradlew :{module}:wire` then grep to locate       │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### Correct Workflow

```
1. Design proto (API / Entity / Constant)
   ↓
2. Run: gradlew :{module}:wire
   ↓
3. wire generates:
   - Controller (src/generated/) ← READ-ONLY
   - Entity (src/generated/) ← READ-ONLY
   - ServiceImpl skeleton (src/main/java/) ← Fill method bodies
   - Trait Repository skeleton (src/main/trait/) ← Add custom queries
   ↓
4. grep to find generated files (if not immediately visible)
   ↓
5. Implement business logic in provided skeletons
```

## 🏗️ Core Architecture

### Backend Layer Flow

```
Proto Entity → wire → [Entity, DSL, Repository] → Trait Extension → ServiceImpl → Controller
```

**Generated artifacts** (`src/generated/domain`):
- `XxxEntity` — Domain entity
- `XxxEntityDSL` — extends `DSL<XxxEntity>`
- `XxxEntityRepository` — extends `HopeJdbc<XxxEntity>`, `ListCrudRepository<XxxEntity, Long>`

**ListCrudRepository provides** (NO need to redefine):
| Method | Description |
|--------|-------------|
| `save(entity)` / `saveAll(entities)` | Insert or update |
| `findById(id)` / `findAll()` / `findAllById(ids)` | Read operations |
| `count()` | Count all |
| `deleteById(id)` / `delete(entity)` | Delete operations |

Reference: https://docs.spring.io/spring-data/commons/reference/repositories/core-concepts.html

### Security Architecture (ApiHug JWT + AOP Pipeline)

> ApiHug uses its **own JWT + AOP pipeline** — NOT Spring Security. All components below are **framework-internal, auto-configured**.

```
HTTP Request
  → JWTFilter (extract JWT → build Customer → HopeContextHolder)
  → Controller
  → SecurityAspect (AOP BEFORE, priority=100)
  → HopeSecurityManager.check() (proto RBAC vs Customer)
  → ServiceImpl (RBAC already enforced, zero auth code)
```

| Internal Component | Class (auto-configured, DO NOT touch)                                                |
|--------------------|--------------------------------------------------------------------------------------|
| JWT Filter         | `hope.common.spring.security.{servlet\|reactive}.jwt.JWTFilter`                      |
| AOP Aspect         | `hope.common.spring.security.SecurityAspect`                                         |
| RBAC Manager       | `hope.common.spring.security.HopeSecurityManager`                                    |
| JWT Provider       | `hope.common.spring.security.JWTTokenProvider`                                       |
| Context Holder     | `hope.common.spring.security.context.HopeContextHolder` ← **only this used in code** |

**CRITICAL**: Permissions defined in proto → AOP enforced at runtime → **ZERO RBAC code** in ServiceImpl/DomainService/Repository

## 🗄️ Repository Layer (Spring Data JDBC)

### 🔌 Entity Wire Interfaces

> Proto `wires: [...]` → generated entity **auto-implements** corresponding interfaces (FQN see [PACKAGE REFERENCE](#package-reference)). All setters return `T` (builder pattern).

| Proto Wire     | Java Interface                    | Getter / Setter                                                                                               |
|----------------|-----------------------------------|---------------------------------------------------------------------------------------------------------------|
| `IDENTIFIABLE` | `Identifiable<T>`                 | `getId()` / `setId(Long)`                                                                                     |
| `AUDITABLE`    | `Auditable<T, Identify>`          | `getCreatedAt()` / `setCreatedAt(LocalDateTime)`, `getCreatedBy()` / `setCreatedBy(Identify)`, `getUpdatedAt()` / `setUpdatedAt(LocalDateTime)`, `getUpdatedBy()` / `setUpdatedBy(Identify)` |
| `DELETABLE`    | `Deletable<T, Identify>`          | `isDeleted()` / `setDeleted(Boolean)`, `getDeletedAt()` / `setDeletedAt(LocalDateTime)`, `getDeletedBy()` / `setDeletedBy(Identify)` |
| `TENANTABLE`   | `Tenantable<T, Tenant>`           | `getTenantId()` / `setTenantId(Tenant)`                                                                       |
| `VERSIONABLE`  | `Versionable<T>`                  | `getVersion()` / `setVersion(Long)`                                                                           |
| `ALL`          | **`Domain<T, Identify, Tenant>`** | All above — `@MappedSuperclass`, implements all 5 interfaces                                                  |

- **`Identify`** = operator identity type (`Long` or `String`); **`Tenant`** = tenant identity type (`Long`)
- **Immutable after insert**: `id`, `createdAt`, `tenantId` — framework enforces `@InsertOnlyProperty`
- **Soft-delete**: `deleted` defaults to `false`; audit fields (`createdAt/By`, `updatedAt/By`) auto-filled by Spring Data

```java
// wires: [IDENTIFIABLE, AUDITABLE, DELETABLE] → entity implements all three
user.setDeleted(true).setDeletedAt(LocalDateTime.now()).setDeletedBy(currentUserId);
```

### ⚠️ CRITICAL: Trait Extension is the ONLY Place for Queries

**ALL query logic MUST be in trait extension — NEVER in service layer.**

```
❌ WRONG: Service assembles EasyCriteria, SQL, or DSL conditions
✅ CORRECT: Service calls repository method; repository implements query logic
```

### Trait Extension Patterns

**Repository Traits:**
- Normal: `com.example.domain.feature.repository.UserRoleRepository`
- Trait: `t.com.example.domain.feature.repository._UserRoleRepository` (package with prefix `t.`, name with prefix `_`) in `src/main/trait/`
- Trait interface is **AUTO-GENERATED by wire** — **FORBIDDEN to create manually**
- **FORBIDDEN** to remove comments in generated trait repository

> **IMPORTANT**: Trait extension inherits ALL methods from `ListCrudRepository` and `HopeJdbc`. **ONLY add methods that don't exist**.

### Pattern 1: Spring Data Query Methods (PREFERRED for simple queries)

Spring Data auto-implements methods based on naming convention:

```java
interface _UserRepository extends UserRepository {
    // === SINGLE FIELD QUERIES ===
    Optional<UserEntity> findByEmail(String email);
    Optional<UserEntity> findByUsernameAndTenantId(String username, Long tenantId);
    List<UserEntity> findByStatus(String status);

    // === COMPARISON OPERATORS ===
    List<UserEntity> findByCreatedAtAfter(Instant date);
    List<UserEntity> findByAgeGreaterThanEqual(Integer age);

    // === STRING MATCHING ===
    List<UserEntity> findByNameContaining(String name);      // LIKE '%name%'
    List<UserEntity> findByNameStartingWith(String prefix);  // LIKE 'prefix%'

    // === IN / NOT IN ===
    List<UserEntity> findByStatusIn(List<String> statuses);
    List<UserEntity> findByRoleNotIn(List<String> roles);

    // === NULL CHECKS ===
    List<UserEntity> findByDeletedAtIsNull();
    List<UserEntity> findByDeletedAtIsNotNull();

    // === BOOLEAN ===
    List<UserEntity> findByActiveTrue();
    List<UserEntity> findByActiveFalse();

    // === ORDERING ===
    List<UserEntity> findByTenantIdOrderByCreatedAtDesc(Long tenantId);

    // === LIMIT / TOP ===
    Optional<UserEntity> findFirstByTenantIdOrderByCreatedAtDesc(Long tenantId);
    List<UserEntity> findTop10ByStatusOrderByCreatedAtDesc(String status);

    // === COUNT / EXISTS ===
    long countByTenantId(Long tenantId);
    boolean existsByEmail(String email);
}
```

| Keyword                            | Example                          | SQL Equivalent                     |
|------------------------------------|----------------------------------|------------------------------------|

### Pattern 2: Default Methods with EasyCriteria (for dynamic queries)

#### Complete Pagination Flow (Proto → ServiceImpl → Repository)

**Step 1: Proto Design** — Enable pagination with `pageable: true`
```proto
rpc SearchUsers (SearchUsersRequest) returns (UserSummary) {
  option (hope.swagger.operation) = {
    post: "/users/search";
    pageable: true;  // ← Framework injects PageRequest
  };
}

message SearchUsersRequest {
  string keyword = 1;  // Business filters ONLY — NO page/size/sort
  int32 status = 2;
}
```

**Step 2: ServiceImpl** — Receive `PageRequest` from framework, pass to repository
```java
@Override
public void searchUsers(PageableResultBuilder<UserSummary> builder,
                        SearchUsersRequest request,
                        PageRequest pageParameter) {  // ← Injected by framework
    // Call repository with PageRequest
    Page<UserEntity> page = userRepo.search(
        request.getTenantId(),
        request.getKeyword(),
        request.getStatus(),
        pageParameter  // ← Pass through
    );

    // Build pageable response
    builder.setPageIndex(pageParameter.getPage())
           .setPageSize(pageParameter.getSize())
           .setTotalCount(page.getTotalElements())
           .setTotalPage(page.getTotalPages())
           .setData(page.getContent());
    // NEVER call done() — controller handles it
}
```

**Step 3: Repository Trait** — Build dynamic criteria, execute paginated query

```java
interface _UserRepository extends UserRepository {
    default Page<UserEntity> search(Long tenantId, String keyword, String status, PageRequest page) {
        // Build criteria
        Criteria criteria = EasyCriteria.eq(_Tenantable_.TENANT_ID, tenantId)
                .and(EasyCriteria.eq(_Deletable_.DELETED, false));

        // Add dynamic filters
        if (keyword != null && !keyword.isBlank()) {
            criteria = criteria.and(EasyCriteria.like(Domain.Name, keyword));
        }
        if (status != null) {
            criteria = criteria.and(EasyCriteria.eq(Domain.Status, status));
        }

        // Execute paginated query
        return findAll(criteria, this.page(page));
    }
}
```

**Flow Summary:**
```
Proto (pageable: true)
   ↓
Controller (auto-injects PageRequest)
   ↓
ServiceImpl (receives PageRequest → passes to repository)
   ↓
Repository (builds Criteria → calls findAll(criteria, page))
   ↓
Page<UserEntity> returned up the chain
```

#### Other Examples

```java
interface _UserRepository extends UserRepository {
    // Exists check (efficient)
    default boolean existsByEmailAndTenant(String email, Long tenantId) {
        return exists(EasyCriteria.eq(Domain.Email, email)
                .and(EasyCriteria.eq(_Tenantable_.TENANT_ID, tenantId)));
    }

    // Soft-delete aware count
    default long countActiveByTenant(Long tenantId) {
        Criteria criteria = EasyCriteria.eq(_Tenantable_.TENANT_ID, tenantId)
                .and(EasyCriteria.eq(_Deletable_.DELETED, false))
                .and(EasyCriteria.eq(Domain.Status, "ACTIVE"));
        return count(criteria);
    }
}
```

**EasyCriteria API Reference:**

| Method                                   | Description                                         |
|------------------------------------------|-----------------------------------------------------|
| `eq(col, val)` / `ne(col, val)`          | Equality / not equal                                |
| `lt(col, val)` / `lte(col, val)`         | Less than / less than or equal                      |
| `gt(col, val)` / `gte(col, val)`         | Greater than / greater than or equal                |
| `between(col, from, to)`                 | Range                                               |
| `like(col, pattern)`                     | LIKE (auto-escapes `%`, `_`), **FORBIDDEN** add '%' |
| `startWith(col, prefix)`                 | LIKE 'prefix%', **FORBIDDEN** add '%'               |
| `in(col, values)` / `notIn(col, values)` | IN / NOT IN                                         |
| `isNull(col)` / `isNotNull(col)`         | Null checks                                         |

> **NULL-SAFE**: All methods return `Criteria.empty()` when value is `null`.

### ⚠️ Performance Anti-Patterns

#### ❌ N+1 Problem
```java
// ❌ WRONG: N+1 queries
List<Order> orders = orderRepo.findByTenantId(tenantId);
for (Order order : orders) {
    User user = userRepo.findById(order.getUserId());  // N queries!
}

// ✅ CORRECT: Batch fetch
List<Order> orders = orderRepo.findByTenantId(tenantId);
Set<Long> userIds = orders.stream().map(Order::getUserId).collect(Collectors.toSet());
Map<Long, User> userMap = userRepo.findByIdIn(userIds).stream()
        .collect(Collectors.toMap(User::getId, Function.identity()));
```

#### ❌ Inefficient Exists Check
```java
// ❌ WRONG
boolean exists = userRepo.findByEmail(email).isPresent();

// ✅ CORRECT
boolean exists = userRepo.existsByEmail(email);
```

#### ❌ Unbounded Result Set
```java
// ❌ WRONG
List<Order> orders = orderRepo.findByTenantId(tenantId);  // Could be millions!

// ✅ CORRECT: Use pagination
Page<Order> orders = orderRepo.findByTenantId(tenantId, PageRequest.of(0, 100));
```

#### ❌ Missing Soft-Delete Filter
```java
// ❌ WRONG: Includes deleted records
List<User> users = userRepo.findByTenantId(tenantId);

// ✅ CORRECT: Filter deleted
List<User> users = userRepo.findByTenantIdAndDeletedFalse(tenantId);
```

#### ❌ In-Memory Pagination
```java
// ❌ WRONG: Fetches ALL rows, then slices in Java
List<User> all = userRepo.findByTenantIdAndDeletedFalse(tenantId);
List<User> page = all.subList(offset, Math.min(offset + size, all.size()));

// ❌ WRONG: Stream skip/limit — still loads everything from DB
List<User> page = userRepo.findAll().stream().skip(offset).limit(size).toList();

// ✅ CORRECT: DB-level pagination
Page<User> page = userRepo.findAll(criteria, PageRequest.of(pageIndex, pageSize));
```

### How RBAC Works (Framework-Driven)

```
Proto Definition                    Runtime Enforcement
─────────────────                  ─────────────────────
authorization: {                   HopeSecurityManager (AOP)
  rbac: {                              ↓
    roles: ["ADMIN"]              ┌─────────────────────┐
    authorities: ["USER_DELETE"]  │ Intercept request   │
  }                               │ Check proto metadata│
                                  │ Allow or Deny       │
                                  └─────────────────────┘
                                         ↓
                                   Controller (no RBAC code)
                                         ↓
                                   ServiceImpl (no RBAC code)
```

### CORRECT Approach

```java
// ✅ CORRECT — Zero RBAC code in implementation
@Service
public class UserServiceImpl implements UserService {

    @Override
```

```java
// ONLY approved way — do NOT pass user via method parameters
{Domain}Customer customer = ({Domain}Customer) HopeContextHolder.customer();
// Customer may be null or anonymous
Long currentUserId = customer.getId();
Long tenantId      = customer.getTenantId();
```

## ❌ Error Handling

### Core Principle: Error-First, Proto-Driven

**Every error that escapes a service method MUST carry:**
- A typed, proto-defined error enum value (with `code`, `title`, `http_status`, `phase`, `severity`)
- Wrapped in `HopeErrorDetailException` OR returned via `builder.error()`

> `HopeErrorDetailException` is a `RuntimeException` — no need to declare or catch. Global `HopeExceptionHandler` (`@ControllerAdvice`) intercepts automatically.

### Error Definition (Proto — MANDATORY First Step)

```proto
syntax = "proto3";
package com.example.order.infra.order;

enum OrderErrorEnum {
  ORDER_NOT_FOUND = 0 [(hope.constant.field) = {
    code: 20001,
    message: "order not found",
    message2: "订单不存在",
    error: {
      title: "Order Not Found",
      http_status: NOT_FOUND,
      phase: SERVICE,
      severity: ERROR
    }
  }];

  ORDER_ALREADY_CANCELLED = 1 [(hope.constant.field) = {
    code: 20002,
    message: "order already cancelled",
    error: { title: "Order Already Cancelled", http_status: CONFLICT, phase: SERVICE, severity: WARN }
  }];
}
```

**After `gradlew :{module}:wire`:**
```java
// AUTO-GENERATED — implements hope.common.api.error.Error
public enum OrderErrorEnum implements Error {
    ORDER_NOT_FOUND(20001, "order not found", HttpStatus.NOT_FOUND, Phase.SERVICE, Severity.ERROR),
    ORDER_ALREADY_CANCELLED(20002, "order already cancelled", HttpStatus.CONFLICT, Phase.SERVICE, Severity.WARN);
}
```

### Throwing Errors in Business Logic

#### DomainService — Always Throw `HopeErrorDetailException`

Domain/infrastructure layers have no access to response builder — MUST throw exception.

```java
public Order cancelOrder(Long orderId) {
    Order order = orderRepository.findById(orderId)
        .orElseThrow(() ->
            HopeErrorDetailException.errorBuilder(OrderErrorEnum.ORDER_NOT_FOUND)
                .setMessage("Order not found: " + orderId)
                .build()
        );

    if (order.isCancelled()) {
        throw HopeErrorDetailException.errorBuilder(OrderErrorEnum.ORDER_ALREADY_CANCELLED).build();
    }
    order.cancel();
    return orderRepository.save(order);
}
```

#### ServiceImpl — Three Valid Patterns

**Pattern A (PREFERRED): Let exception propagate**
```java
@Override
public void cancelOrder(SimpleResultBuilder<Void> builder, Long orderId) {
    orderDomainService.cancelOrder(orderId);  // throws HopeErrorDetailException if error
    builder.ok("Order cancelled");
    // HopeExceptionHandler intercepts automatically
}
```

**Pattern B: Catch only when adding context**
```java
@Override
public void cancelOrder(SimpleResultBuilder<Void> builder, Long orderId) {
    try {
        orderDomainService.cancelOrder(orderId);
        builder.ok("Order cancelled");
    } catch (HopeErrorDetailException e) {
        builder.badRequest().errors(e.getErrors());  // forward to builder
    }
}
```

**Pattern C: Use builder directly for simple checks**
```java
@Override
public void getOrder(SimpleResultBuilder<OrderDetail> builder, Long orderId) {
    Order order = orderRepository.findById(orderId).orElse(null);
    if (order == null) {
        builder.notFound().error(OrderErrorEnum.ORDER_NOT_FOUND);
        return;
    }
    builder.payload(orderMapper.toDetail(order));
}
```

#### Repository/Infrastructure — Throw `HopeErrorDetailException`

```java
public Order requireById(Long id) {
    return findById(id).orElseThrow(() ->
        HopeErrorDetailException.errorBuilder(OrderErrorEnum.ORDER_NOT_FOUND)
            .setMessage("Order ID: " + id)
            .build()
    );
}
```

### `HopeErrorDetailException` API

| Method                             | Usage                                  |
|------------------------------------|----------------------------------------|
| `errorBuilder(ErrorEnum)`          | Start with proto error **(PREFERRED)** |
| `errorBuilder()`                   | Empty builder (add errors manually)    |
| `errorBuilder(message, ErrorEnum)` | Message + proto error                  |
| `noImplement()`                    | Not-yet-implemented features           |

```java
HopeErrorDetailException.errorBuilder(OrderErrorEnum.ORDER_NOT_FOUND)
    .setMessage("Human-readable message for logs")   // optional
    .setCause(originalException)                       // optional
    .build();
```

> **Prefer enum-based factory over manual `ErrorBuilder`** — manual construction loses proto metadata traceability.

### `builder.error()` in ServiceImpl

```java
builder.notFound().error(OrderErrorEnum.ORDER_NOT_FOUND);
builder.badRequest().error(OrderErrorEnum.ORDER_PAYMENT_FAILED);
builder.conflict().error(OrderErrorEnum.ORDER_ALREADY_CANCELLED);
```

> HTTP status helper (`.notFound()`, `.badRequest()`) sets transport status. `.error(ErrorEnum)` encodes structured payload.

### Validation Errors — Framework Responsibility

**Input validation is handled automatically** by [Spring Bean Validation](https://docs.spring.io/spring-framework/reference/core/validation/beanvalidation.html) — **DO NOT write manual validation in ServiceImpl.**

| Validation Type                 | Handler                     | Developer Action                 |
|---------------------------------|-----------------------------|----------------------------------|
| Proto field constraints         | Framework auto-validates    | Define constraints in proto      |
| Missing required fields         | Framework returns 400       | None                             |
| Custom business rule validation | DomainService / ServiceImpl | Throw `HopeErrorDetailException` |

```proto
// ✅ Define validation in proto — framework enforces
message CreateOrderRequest {
  string product_id = 1 [(hope.swagger.field) = { required: true }];
  int32 quantity = 2 [(hope.swagger.field) = { minimum: 1, maximum: 9999 }];
}
```

**FORBIDDEN:**
```java
// ❌ Do NOT manually validate proto-constrained fields
if (request.getProductId() == null) {
    builder.badRequest().message("Product ID required");
    return;
}
```

### Forbidden Patterns

```java
// ❌ Raw RuntimeException with string message
throw new RuntimeException("Order not found");

// ❌ Ad-hoc Error object with hardcoded codes
Error error = new Error().setCode(404).setDescription("not found");
builder.badRequest().error(error);

// ❌ Catching and swallowing HopeErrorDetailException
try { domainService.process(); }
catch (HopeErrorDetailException e) { logger.error("error", e); }  // structured errors lost

// ❌ Wrapping in another RuntimeException
throw new RuntimeException("wrapped", hopeException);  // loses all metadata

// ❌ Unnecessary try-catch that just re-throws
try { domainService.process(); } catch (HopeErrorDetailException e) { throw e; }  // pointless

// ❌ Manual input validation for proto-defined constraints
if (request.getName() == null) { builder.badRequest().message("Name required"); return; }
```

### Error Code Namespace Convention

**Error codes MUST be unique within the domain module.**

```proto
// ✅ CORRECT — unique codes across enums in same domain
enum UserErrorEnum {
  USER_NOT_FOUND      = 0 [(hope.constant.field) = {code: 1001, ...}];
  USER_ALREADY_EXISTS = 1 [(hope.constant.field) = {code: 1002, ...}];
}

enum OrderErrorEnum {
  ORDER_NOT_FOUND = 0 [(hope.constant.field) = {code: 2001, ...}];  // different range
}

// ❌ WRONG — duplicate codes across enums
enum PaymentErrorEnum {
  PAYMENT_FAILED = 0 [(hope.constant.field) = {code: 1001, ...}];  // collides with UserErrorEnum
}
```

## 📊 Result & Pageable

### Core Data Structures

#### `Result<T>` – Unified API Response

| Field     | Type          | Description                       |
|-----------|---------------|-----------------------------------|
| `code`    | `int`         | `0` = success, non‑zero = failure |
| `message` | `String`      | Human‑readable message            |
| `data`    | `T`           | Payload (generic)                 |
| `errors`  | `List<Error>` | Detailed error list (optional)    |

#### `Pageable<T>` – Paginated Result

| Field        | Type      | Description                 |
|--------------|-----------|-----------------------------|
| `pageIndex`  | `int`     | Current page **⚠️ 0‑based** |
| `pageSize`   | `int`     | Items per page              |
| `totalCount` | `long`    | Total records               |
| `totalPage`  | `int`     | Total pages                 |
| `data`       | `List<T>` | Items for current page      |

#### `PageRequest` – Pagination Parameters

| Field  | Type           | Description               | Constraints     |
|--------|----------------|---------------------------|-----------------|
| `page` | `Integer`      | Page **⚠️ 0‑based**       | Min 0           |
| `size` | `Integer`      | Items per page            | Min 1, Max 1024 |
| `sort` | `List<String>` | Sort: `"field,direction"` | Optional        |

> **Critical:** All page indexes are **0‑based** (aligns with Spring Data).

### Proto Contract Rules — CRITICAL

#### Quick Decision Table

| Scenario              | Proto Setting           | Request           | Response         |
|-----------------------|-------------------------|-------------------|------------------|
| Large dataset, paging | `pageable: true`        | Query fields only | Single item type |
| Small list, no paging | `output_repeated: true` | Normal            | Single item type |
| Batch input           | `input_repeated: true`  | Single item type  | Normal           |
| Batch I/O             | Both                    | Single item type  | Single item type |

#### Paginated RPC (`pageable: true`)

```proto
// ✅ CORRECT
rpc SearchUsers (SearchUsersRequest) returns (UserSummary) {
  option (hope.swagger.operation) = {
    get: "/users";
    pageable: true;   // framework injects page/size/sort
  };
}

message SearchUsersRequest {
  string keyword = 1;
  // ❌ NEVER add page/size/sort here
}

// ❌ WRONG — wrapping response yourself
rpc SearchUsers (SearchUsersRequest) returns (PageableUserSummary);  // ❌ double-wrapping
```

**Generated ServiceImpl signature:**
```java
void searchUsers(PageableResultBuilder<UserSummary> builder,
                 SearchUsersRequest request,
                 PageRequest pageParameter);
```

#### List Output (`output_repeated: true`)

For simple list responses **without pagination**, use `output_repeated: true`.

```proto
// ✅ CORRECT
rpc ListRoles (ListRolesRequest) returns (RoleSummary) {
  option (hope.swagger.operation) = {
    get: "/roles";
    output_repeated: true;  // framework wraps in Result<List<RoleSummary>>
  };
}

// ❌ WRONG
rpc ListRoles (ListRolesRequest) returns (RoleSummaryList);  // ❌ unnecessary wrapper
```

**Generated ServiceImpl signature:**
```java
void listRoles(SimpleResultBuilder<List<RoleSummary>> builder, ListRolesRequest request);
```

#### List Input (`input_repeated: true`)

For batch operations accepting **a list of items as input**, use `input_repeated: true`.

```proto
// ✅ CORRECT
rpc BatchCreateUsers (CreateUserRequest) returns (UserSummary) {
  option (hope.swagger.operation) = {
    post: "/users/batch";
    input_repeated: true;   // framework unwraps into List<CreateUserRequest>
    output_repeated: true;  // returns List<UserSummary>
  };
}

message CreateUserRequest {
  string username = 1;
  string email = 2;
}

// ❌ WRONG
rpc BatchCreateUsers (BatchCreateUsersRequest) returns (UserSummaryList);  // ❌ unnecessary wrappers
```

**Generated ServiceImpl signature:**
```java
void batchCreateUsers(SimpleResultBuilder<List<UserSummary>> builder,
                      List<CreateUserRequest> requests);  // List<T> injected
```

### Service Implementation Patterns

#### SimpleResultBuilder

```java
@Override
public void getUser(SimpleResultBuilder<UserDetail> builder, Long id) {
    User user = userRepository.findById(id).orElse(null);
    if (user == null) {
        builder.notFound().error(UserErrorEnum.USER_NOT_FOUND);
        return;
    }
    builder.payload(userMapper.toDetail(user));
    // NEVER call done() — controller handles it
}
```

**HTTP status helpers:**
```java
builder.notFound().error(UserErrorEnum.USER_NOT_FOUND);
builder.badRequest().error(OrderErrorEnum.ORDER_PAYMENT_FAILED);
builder.conflict().error(OrderErrorEnum.ORDER_ALREADY_CANCELLED);
```

#### PageableResultBuilder

```java
@Override
public void searchUsers(PageableResultBuilder<UserSummary> builder,
                         QueryUsersRequest request,
                         PageRequest pageParameter) {
    Page<User> page = userRepository.findAll(
        buildCriteria(request),
        org.springframework.data.domain.PageRequest.of(
            pageParameter.getPage(), pageParameter.getSize()
        )
    );

    builder.setPageIndex(pageParameter.getPage())
           .setPageSize(pageParameter.getSize())
           .setTotalCount(page.getTotalElements())
           .setTotalPage(page.getTotalPages())
           .setData(page.getContent());
    // NEVER call done()
}
```

### PageRequest Utilities

```java
PageRequest.empty();                    // default: page=0, size=20
PageRequest.immutableOf(0, 50);         // immutable instance
new PageRequest().setPage(0).setSize(20).setSort(List.of("createTime,desc"));
```

> `PageRequest` is guarded by `PageRequestGuardian` in generated controllers.

### ⚠️ FORBIDDEN Patterns (ZERO TOLERANCE)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  PAGEABLE API — ABSOLUTELY FORBIDDEN                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│  ❌ message SearchRequest { int32 page = 1; int32 size = 2; }              │
│  ❌ rpc Search (...) returns (PageableUserSummary);                        │
│  ❌ rpc Search (...) returns (UserSummaryList);                            │
│  ❌ builder.done() in service implementation                               │
│  ❌ return null for Pageable.data (return empty list instead)              │
│  ❌ In-memory pagination (subList/skip/limit on full query result)        │
└─────────────────────────────────────────────────────────────────────────────┘
```

| # | Rule                                              | Why                                        |
|---|---------------------------------------------------|--------------------------------------------|
| 1 | **NEVER call `done()`**                           | Generated controller handles it            |
| 2 | **NEVER add `page/size/sort`** to Request Message | Framework injects via `pageable: true`     |
| 3 | **NEVER wrap response** in list/pageable type     | Framework wraps `Result<Pageable<T>>`      |
| 4 | **NEVER return `null`** for `Pageable.data`       | Return empty list instead                  |
| 5 | **NEVER paginate in memory**                      | `subList`/`skip`/`limit` = full table scan |

## 📋 Quick Reference

### Repository Best Practices

| Practice                                              | Why                                                 |
|-------------------------------------------------------|-----------------------------------------------------|
| **Use Query Methods for simple queries**              | Auto-implemented, no code needed                    |
| **Use EasyCriteria for dynamic queries**              | Type-safe, null-safe                                |
| **Use `exists()` instead of `findOne().isPresent()`** | Efficient EXISTS query                              |
| **Always add pagination for list queries**            | Prevent unbounded result sets                       |
| **Filter soft-delete in queries**                     | `deleted = false`                                   |
| **Define SQL constants in `_SQL` interface**          | Centralized, maintainable                           |
| **Run `wire` after proto changes**                    | Keep DSL/repository in sync                         |
| **NEVER write query logic in service**                | All queries in trait extension                      |
| **NEVER redefine ListCrudRepository methods**         | save/findById/findAll/delete already provided       |
| **NEVER redefine HopeJdbc methods**                   | findOne/exists/count with Criteria already provided |

### Error Handling Quick Reference

| Scenario                                 | Pattern                                                                |
|------------------------------------------|------------------------------------------------------------------------|
| Entity not found in DomainService        | `throw HopeErrorDetailException.errorBuilder(ENUM).build()`            |
| Business rule violation in DomainService | `throw HopeErrorDetailException.errorBuilder(ENUM).build()`            |
| Not found check in ServiceImpl           | `builder.notFound().error(ENUM)`                                       |
| Forward DomainService error to builder   | `catch(HopeErrorDetailException e) { builder.errors(e.getErrors()); }` |
| Input validation                         | Define in proto — framework handles                                    |
| Unexpected system error                  | Let propagate — `HopeExceptionHandler` catches all `Throwable`         |

### Security Quick Reference

| Do                                   | Don't                                        |
|--------------------------------------|----------------------------------------------|
| Define RBAC in proto `authorization` | Write security checks in ServiceImpl         |
| Use `HopeContextHolder.customer()`   | Pass user via method parameters              |
| Let framework handle JWT             | Write JWT logic manually                     |
| Use proto `rbac.authorities`         | Use `@PreAuthorize` / `@Secured` annotations |
| Implement 5 security components once | Add security annotations to business logic   |

### Pageable Quick Reference

| Do                                    | Don't                                 |
|---------------------------------------|---------------------------------------|
| Use `pageable: true` in proto         | Add page/size/sort to Request Message |
| Return single item type               | Wrap response in Pageable/List type   |
| Use 0-based page index                | Forget page index is 0-based          |
| Chain builder methods                 | Call `builder.done()`                 |
| Return empty list for null data       | Return `null` for `Pageable.data`     |
| Use `output_repeated: true` for lists | Create wrapper message types          |
