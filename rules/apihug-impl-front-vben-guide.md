# APIHug Frontend Vben Guide

## 0. Core Constraints (MUST / MUST NOT)

> **Violating these rules causes code conflicts or maintenance disasters.**

| Constraint                                                              | Reason                                                       |
|-------------------------------------------------------------------------|--------------------------------------------------------------|
| **NEVER** manually create `router/routes/modules/*.ts`                  | Auto-generated by `gradlew :{frontend_ui_module}:ui`         |
| **NEVER** manually maintain `menus/index.ts` or `menu.json`             | Auto-derived from route tree                                 |
| **NEVER** write raw HTTP requests (`fetch` / `axios` / `requestClient`) | Use generated Service SDK only                               |
| **NEVER** wrap or extend the HTTP client yourself                       | `request.ts` handles Token, 401 refresh, errors             |
| **AVOID** hardcoding route path strings                                 | Paths derived from directory structure                       |
| **AVOID** duplicate `defineOptions.name` values                         | Breaks `keepAlive` caching                                   |
| **NEVER** write raw `WebSocket(...)` or `EventSource(...)`              | Use `WebSocketClient`/`SSEClient` from `@hope/realtime`      |
| **NEVER** call `configureApiClient`/`configureRealtimeClient` twice     | IoC containers are single-use initialization                 |

## 2. Auto Router Generation

### 2.1 Trigger

```bash
gradlew :{frontend_ui_module}:ui
```

Scans `views/**/*.vue` → generates `router/routes/modules/{module}.ts` + `menus/`.

### 2.2 defineOptions

```vue
<script setup lang="ts">
defineOptions({
  name: 'SystemUser',
  meta: {
    title: 'pages.system.user.title',   // i18n key (auto-wrapped with $t())
    icon: 'lucide:users',
    order: 10,
    authority: ['admin', 'user-mgr'],
    hideInMenu: false,
    keepAlive: true,
    layout: 'BasicLayout',
  },
});
</script>
```

### 2.3 Directory = Route

```
views/
└── system/                     → system.ts
    ├── index.vue               → /system
    ├── role.vue                → /system/role
    └── user/[id]/index.vue     → /system/user/:id
```

- `[param]` → `:param` (required)
- `[[param]]` → `:param?` (optional)

### 2.4 meta Reference

| Field           | Type     | Description                    |
|-----------------|----------|--------------------------------|
| `title`         | String   | Page title (i18n key)          |
| `icon`          | String   | Menu icon (iconify)            |
| `order`         | Number   | Sort order (lower = first)     |
| `authority`     | String[] | Required permission roles      |
| `hideInMenu`    | Boolean  | Hide from menu                 |
| `hideInTab`     | Boolean  | Hide from tab bar              |
| `keepAlive`     | Boolean  | Enable component cache         |
| `layout`        | String   | `BasicLayout` / `AuthPageLayout` |
| `noBasicLayout` | Boolean  | No base layout (login/error)   |
| `ignoreAccess`  | Boolean  | Skip permission check          |
| `path`          | String   | Override auto-inferred path    |

## 3. HTTP Client & API SDK

### 3.1 Architecture (IoC Container)

`@hope/api-core` uses an IoC singleton. The app creates a `RequestClient` (from `@vben/request`), wraps it as an `ApiClient` adapter, then injects it once at startup.

```typescript
// bootstrap.ts — from admin-center template
import { configureApiClient, type ApiClient } from '@hope/api-core';
import { requestClient } from '#/api/request';

const apiClientAdapter: ApiClient = {
  get: (url, config) => requestClient.get(url, config),
  post: (url, data, config) => requestClient.post(url, data, config),
  put: (url, data, config) => requestClient.put(url, data, config),
  delete: (url, data, config) => requestClient.delete(url, data, config),
  request: (url, config) => requestClient.request(url, config),
};

configureApiClient(apiClientAdapter);
```

> After injection, all `useGet`/`usePost`/`useUpload` calls in generated SDK automatically use this client.

### 3.2 Interceptor Chain (Three-Layer)

The `RequestClient` in `api/request.ts` has a three-layer response interceptor chain:

```
1. Unified Format (fulfilled) — unwraps { code, data, message, errors }
2. Auth Handler  (rejected)  — 401 → token refresh + request queue retry
3. Error Message (rejected)  — network/timeout/HTTP errors → ant-design message
```

Key behaviors:
- `code === 0 || 200` → return `data` field
- `errors[]` non-empty → throw with error details
- 401 → refresh token via `refreshTokenApi()`, queue pending requests, retry all with new token
- Token refresh failure → clear token, redirect to login (or show login modal)

> **NEVER** duplicate these interceptors — they are already in the template.

### 3.3 SDK Origin

Auto-generated by `gradlew :{frontend_ui_module}:ui` (`frontend_ui_module` MUST have `uiFrom project(':{app_module}')` in its `build.gradle`):

```
@{scope}/{app_module}-sdk/
  ├── index.ts         ← Service entry point (re-exports all services)
  └── schema/request/  ← TypeScript type definitions from proto
```

> `{scope}` is the npm scope from `hope-wire.json` → `ui.scope`. `{app_module}` is the backend module name.

### 3.4 Helper Functions

```typescript
import { useGet, usePost, usePut, useDelete, useUpload, pathToUrl } from '@hope/api-core';

// Typed HTTP helpers (used inside generated SDK)
const getData = useGet<ResponseType, ParamsType>();
const postData = usePost<ResponseType, BodyType, ParamsType>();

// File upload (auto multipart/form-data)
const upload = useUpload<ResponseType, { file: File; name: string }>();

// Path parameter substitution: /api/{id}/detail → /api/123/detail
const url = pathToUrl('/api/{id}/detail/{type}', '123', 'user');
```

### 3.5 Core Types

```typescript
interface ResponseBody<T> { code: number; data: T; message: string; errors?: ApiError[]; }
interface PageRequest { page: number; size: number; sort?: SortField[]; }
interface PageableResult<T> { pageIndex: number; pageSize: number; totalCount: number; totalPage: number; data: T[]; }
interface RequestConfigExtra { anonymous?: boolean; loading?: boolean; lowLimitRiskyMode?: string; }
```

### 3.6 Import and Call

> Replace `{scope}` with your npm scope and `{app_module}` with the backend module name.

```typescript
import { XxxService } from '@{scope}/{app_module}-sdk';

// Paginated query (page is 0-based)
const res = await XxxService.searchXxx({}, { page: currentPage - 1, size: pageSize });

// List query
const res = await XxxService.listXxx({});

// CRUD
await XxxService.addXxx(values);
await XxxService.updateXxx(values);
await XxxService.deleteXxx(id);
```

> **Page index is 0-based.** vxe-table's `currentPage` starts at 1, so subtract 1.

### 3.7 Forbidden

```typescript
// ❌ NEVER — use generated Service SDK
import { requestClient } from '#/api/request';
const data = await requestClient.get('/api/xxx');

// ❌ NEVER — use Service SDK
fetch('/api/xxx').then(...);

// ❌ NEVER — already done in bootstrap.ts
configureApiClient(anotherClient);
```

### 3.8 Error Handling Convention

The interceptor chain in `request.ts` handles most errors globally. For business-level error handling:

```typescript
try {
  await XxxService.addXxx(values);
  message.success($t('page.common.saveSuccess'));
} catch (error: any) {
  // error.errors[] contains ApiError details from backend
  if (error?.errors?.length) {
    // Show field-level validation errors
    error.errors.forEach((e: any) => message.error(e.message));
  }
  // Other errors are already handled by global interceptor — do NOT duplicate
}
```

## 4. Real-time Communication

`@hope/realtime` provides SSE and WebSocket clients with the same IoC pattern as api-core.

### 4.1 Configuration

```typescript
// bootstrap.ts — alongside configureApiClient (template has this commented out)
import { configureRealtimeClient } from '@hope/realtime';
import { useAppConfig } from '@vben/hooks';
import { useAccessStore } from '@vben/stores';
import { preferences } from '@vben/preferences';

const { apiURL } = useAppConfig(import.meta.env, import.meta.env.PROD);

configureRealtimeClient({
  baseURL: apiURL,
  getHeaders: () => ({
    Authorization: `Bearer ${useAccessStore().accessToken}`,
    'Accept-Language': preferences.app.locale,
  }),
});
```

> Uncomment the realtime block in `bootstrap.ts` when your app needs SSE or WebSocket.

### 4.2 SSE Client (Streaming)

```typescript
import { SSEClient, createNDJSONParser } from '@hope/realtime';

const client = new SSEClient('/api/chat/stream', {
  method: 'POST',
  format: 'ndjson',       // 'ndjson' | 'sse'
  reconnect: { enabled: true, maxAttempts: 3, backoff: 'exponential' },
});

for await (const chunk of client.stream(requestBody, {
  parser: createNDJSONParser<ChatResponse>(),
  onError: (err) => console.error(err),
})) {
  appendMessage(chunk);
}

client.close();
```

> SSE uses `fetch` + `ReadableStream` internally — supports POST with body (unlike native `EventSource`).

### 4.3 WebSocket Client

```typescript
import { WebSocketClient } from '@hope/realtime';

const client = new WebSocketClient('/api/ws/notifications', {
  heartbeat: { enabled: true, interval: 30000 },
  reconnect: { enabled: true, maxAttempts: 5, backoff: 'exponential' },
});

const channel = await client.connect<SendType, ReceiveType>();

channel.on('message', (data: ReceiveType) => handleMessage(data));
channel.on('error', (err) => handleError(err));
channel.on('close', () => handleDisconnect());

channel.send({ type: 'subscribe', topic: 'orders' });
channel.close();
```

### 4.4 Parsers

| Parser | Format | Use Case |
|--------|--------|----------|
| `createNDJSONParser<T>()` | Newline-delimited JSON | AI chat streams, log tailing |
| `createSSEParser<T>()` | `data:` prefixed SSE | Standard SSE event streams |

### 4.5 Reconnect & Heartbeat

| Option | SSE Default | WS Default | Description |
|--------|------------|------------|-------------|
| `reconnect.maxAttempts` | `3` | `5` | Max retry count |
| `reconnect.backoff` | `exponential` | `exponential` | `linear` \| `exponential` |
| `reconnect.delay` | `1000ms` | `1000ms` | Base delay |
| `reconnect.maxDelay` | `30000ms` | `30000ms` | Delay cap |
| `heartbeat.interval` | — | `30000ms` | Ping interval (WS only) |

### 4.6 Forbidden

```typescript
// ❌ NEVER — use SSEClient
const source = new EventSource('/api/stream');

// ❌ NEVER — use WebSocketClient
const ws = new WebSocket('ws://localhost:8080');

// ❌ NEVER — SSEClient handles fetch internally
const res = await fetch('/api/stream');
const reader = res.body.getReader();
```

## 5. Page Patterns

### 5.1 LIST_PAGE

**Path**: `views/{module}/index.vue`

```vue
<script lang="ts" setup>
defineOptions({
  name: 'XxxList',
  meta: { title: 'pages.xxx.title', icon: 'lucide:list', order: 10 },
});

import type { VxeGridProps } from '#/adapter/vxe-table';
import { Page } from '@vben/common-ui';
import { $t } from '@vben/locales';
import { Button, Space, message, Modal } from 'ant-design-vue';
import { useVbenVxeGrid } from '#/adapter/vxe-table';
import { XxxService } from '@{scope}/{app_module}-sdk';

const gridOptions: VxeGridProps = {
  columns: [
    { type: 'seq', width: 60, title: $t('page.common.seq') },
    { field: 'name', minWidth: 150, title: $t('pages.xxx.name') },
    { field: 'createdAt', width: 160, title: $t('page.common.createdAt'),
      formatter: ({ cellValue }) => cellValue ? new Date(cellValue).toLocaleString() : '-' },
    { field: 'action', fixed: 'right', slots: { default: 'action' }, width: 180 },
  ],
  proxyConfig: {
    ajax: {
      query: async ({ page }) => {
        const res = await XxxService.searchXxx({}, { page: page.currentPage - 1, size: page.pageSize });
        return { items: res?.items ?? [], total: res?.totalCount ?? 0 };
      },
    },
  },
  pagerConfig: { enabled: true },
  toolbarConfig: { refresh: true, zoom: true, custom: true },
};

const [Grid, gridApi] = useVbenVxeGrid({
  formOptions: {
    schema: [{ component: 'Input', fieldName: 'keyword', label: $t('page.common.keyword') }],
  },
  gridOptions,
});

function handleDelete(row: any) {
  Modal.confirm({
    title: $t('page.common.confirmDelete'),
    okType: 'danger',
    async onOk() {
      await XxxService.deleteXxx(row.id);
      message.success($t('page.common.deleteSuccess'));
      gridApi.reload();
    },
  });
}
</script>

<template>
  <Page auto-content-height>
    <Grid>
      <template #toolbar-tools>
        <Button type="primary">{{ $t('page.common.add') }}</Button>
      </template>
      <template #action="{ row }">
        <Space>
          <Button size="small" type="link">{{ $t('page.common.edit') }}</Button>
          <Button size="small" danger type="link" @click="handleDelete(row)">{{ $t('page.common.delete') }}</Button>
        </Space>
      </template>
    </Grid>
  </Page>
</template>
```

### 5.2 FORM_PAGE

**Path**: `views/{module}/components/XxxFormModal.vue`

```vue
<script setup lang="ts">
import { useVbenModal } from '@vben/common-ui';
import { useVbenForm } from '#/adapter/form';
import { XxxService } from '@{scope}/{app_module}-sdk';

const [Form, formApi] = useVbenForm({
  schema: [
    { component: 'Input', fieldName: 'name', label: 'Name', rules: 'required' },
    { component: 'Select', fieldName: 'status', label: 'Status' },
  ],
  showDefaultActions: false,
});

const [Modal, modalApi] = useVbenModal({
  connectedComponent: Form,
  async onConfirm() {
    const values = await formApi.validateAndSubmitForm();
    modalApi.lock();
    try {
      values.id ? await XxxService.updateXxx(values) : await XxxService.addXxx(values);
      modalApi.close();
    } finally { modalApi.unlock(); }
  },
});

defineExpose({ Modal });
</script>

<template>
  <Modal title="Add / Edit"><Form /></Modal>
</template>
```

**Parent call:**
```typescript
modalRef.value?.modalApi.open();                    // create
modalRef.value?.modalApi.setData(row).open();       // edit
```

### 5.3 DETAIL_PAGE

**Path**: `views/{module}/[id]/index.vue`

```vue
<script setup lang="ts">
defineOptions({ name: 'XxxDetail', meta: { title: 'pages.xxx.detail', hideInMenu: true } });

import { ref, onMounted } from 'vue';
import { useRoute } from 'vue-router';
import { XxxService } from '@{scope}/{app_module}-sdk';

const route = useRoute();
const detail = ref(null);

onMounted(async () => {
  detail.value = await XxxService.getXxx(Number(route.params.id));
});
</script>
```

## 6. Internationalization (i18n)

All user-facing text MUST use i18n keys:

```typescript
// ✅ Correct
title: $t('pages.system.user.title')
message.success($t('page.common.deleteSuccess'))

// ❌ NEVER hardcode
title: 'User Management'
message.success('Deleted')
```

- i18n keys are defined in `apps/{app}/src/locales/{lang}/`
- Use `page.common.*` for shared labels (add, edit, delete, search, etc.)
- Use `pages.{module}.*` for module-specific labels

## 7. Vben Component API

### useVbenVxeGrid

```typescript
const [Grid, gridApi] = useVbenVxeGrid({ gridOptions, formOptions });

gridApi.reload();            // Reset to page 1 and refresh
gridApi.query();             // Refresh (keep current page)
gridApi.setLoading(true);
gridApi.toggleSearchForm();
```

Return format: `{ items: T[], total: number }`

### useVbenModal

```typescript
const [Modal, modalApi] = useVbenModal({ onConfirm, connectedComponent });

modalApi.open() / .close();
modalApi.lock() / .unlock();    // Lock during submission
modalApi.setData(data) / .getData<T>();
```

### useVbenForm

```typescript
const [Form, formApi] = useVbenForm({ schema });

await formApi.validateAndSubmitForm();
await formApi.setValues(record);   // Populate for edit
await formApi.resetForm();
formApi.updateSchema([...]);
```

## 8. Development Workflow

```
1. Bootstrap: configureApiClient() + configureRealtimeClient() — once at app startup

2. Create views/**/*.vue with defineOptions({ name, meta })

3. Import Service from @{scope}/{app_module}-sdk — NEVER write raw HTTP

4. All user-facing text via $t() — NEVER hardcode strings

5. For real-time: use SSEClient (streaming) or WebSocketClient (bidirectional)

6. Build UI with useVbenVxeGrid / useVbenForm / useVbenModal

7. Run: gradlew :{frontend_ui_module}:ui → generates router + menus + SDK

8. Start dev server → verify routing, permissions, and real-time connections
```
